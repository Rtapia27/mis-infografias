El Paradigma "One-Shot"

Estamos ante un cambio de juego. Gracias a Gemini 3.1 Pro, ya no hablamos de generar plantillas genéricas, sino de "planos digitales" completos. El flujo consiste en dar una instrucción masiva y estructurada (testamento de prompt) que define no solo la estética, sino el comportamiento dinámico del sitio.

1. Anatomía del Prompt Maestro

1 Definición de Rol

Se le ordena a la IA actuar como un "Tecnólogo Creativo Senior" e "Ingeniero Líder de Frontend". Esto eleva la calidad del código y la obsesión por el detalle.

2 Estilo Visual

No basta con decir "bonito". Se usan términos como "estética cinematográfica", "clínica boutique" o "tecnología orgánica" para guiar el motor de diseño.

3 Comportamiento

Se especifican animaciones SVG, efectos de scroll (desenfoques suaves) y micro-transiciones que dan la sensación de producto premium.

2. El Sistema de 6 Preguntas

Para no editar prompts larguísimos, el video propone un sistema de entradas clave que Gemini utiliza para construir la landing:

1. Nombre del Proyecto: [Ej. Vela Arquitectura] 2. ¿Qué vendes/haces?: [Servicios, productos, comunidad] 3. Cliente Ideal: [Perfil específico de usuario] 4. Oferta de Valor: [Qué obtienen exactamente] 5. Prueba Social: [Datos de confianza, proyectos entregados] 6. Llamada a la Acción (CTA): [Ej. Reservar evaluación]

3. Casos de Aplicación Analizados

Clinic Balance

Enfoque moderno en nutrición y estrés. El diseño genera módulos que parecen interfaces de telemetría médica en vivo.

Vela Arquitectura

Estudio premium. Tipografías minimalistas de gran tamaño y secciones separadas por fases de obra (Concepto, Planos, Renders).

Advertencia Técnica: Aunque Gemini 3.1 genera código funcional, siempre se debe revisar la integración de imágenes de fondo y los enlaces de los botones antes de pasar a producción.

4. Flujo de Despliegue Automático

Para que tu sitio sea público, sigue estos pasos:

  1. Copia el código generado en un archivo index.html.
  2. Súbelo a un repositorio de GitHub llamado mis-infografias.
  3. Activa GitHub Pages en la configuración del repo.
